# Code of conduct

## Our standards

Participation in this public project must remain respectful, constructive and focused on the technical work. Expected behavior includes:

- giving and receiving technical feedback professionally;
- respecting different levels of experience, backgrounds and viewpoints;
- keeping public reports free of credentials and private infrastructure details;
- accepting maintainer decisions about project scope and coordinated disclosure.

Harassment, discrimination, threats, publication of another person's private information and deliberately disruptive behavior are not acceptable in project spaces or when representing the project elsewhere.

## Enforcement

Use GitHub's built-in **Report abuse or spam** control for sensitive conduct violations that occur on GitHub. A non-sensitive moderation request may be opened as a repository issue, but it must not contain private personal information. This project does not currently advertise a private conduct inbox. Security vulnerabilities must instead use GitHub private vulnerability reporting as described in `SECURITY.md`.

Include enough context for the maintainer to understand the incident, but do not redistribute sensitive personal information. Reports will be handled as confidentially as practical.

The maintainer may edit or remove contributions, comments, issues or other participation that violates these standards, and may temporarily or permanently restrict participation when necessary. Enforcement decisions are made in the interests of the project and affected community members.
