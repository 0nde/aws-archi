# Code of conduct

## Our standards

Participation in this project must remain respectful, constructive and focused on the technical work. Examples of expected behavior include:

- giving and receiving technical feedback professionally;
- respecting different levels of experience and viewpoints;
- keeping public reports free of credentials and private infrastructure details;
- accepting maintainer decisions about project scope and security disclosure.

Harassment, discrimination, threats, publication of another person's private information and deliberately disruptive behavior are not acceptable.

## Enforcement

Report conduct concerns privately to the repository maintainer through GitHub. Security vulnerabilities must instead use GitHub private vulnerability reporting as described in `SECURITY.md`.

The maintainer may edit or remove contributions, comments, issues or other participation that violates these standards, and may temporarily or permanently restrict participation when necessary.
