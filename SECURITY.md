# Security policy

## Supported versions

Security fixes are applied to the current `latest` image and the newest release line. Older image tags are retained only when explicitly documented and should not be used for new environments.

## Reporting a vulnerability

Use GitHub private vulnerability reporting for suspected vulnerabilities. Do not open a public issue for an unpatched vulnerability or include credentials, tokens, private infrastructure details or exploit data in public discussions.

Include the affected image digest, architecture, reproduction steps, expected impact and any relevant scanner output. Reports will be acknowledged as soon as practical and coordinated disclosure is preferred.

Never submit real AWS, GitHub or registry credentials as test data.
